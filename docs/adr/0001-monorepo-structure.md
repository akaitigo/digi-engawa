# ADR-0001: モノレポ構成（web/ + api/）

## ステータス
承認

## コンテキスト
フロントエンド（Next.js）とバックエンド（Go）を同一リポジトリで管理するか、別リポジトリにするかの判断が必要。MVPフェーズでは開発速度とデプロイの簡便さを優先する。

## 決定
モノレポ構成（web/ + api/ のディレクトリ分離）を採用する。

## 選択肢
| 選択肢 | 長所 | 短所 |
|--------|------|------|
| A: モノレポ（web/ + api/） | 変更の一貫性、PR単位での追跡容易、CI/CD統合 | ビルド複雑化、独立デプロイ制約 |
| B: マルチレポ（別リポジトリ） | 独立デプロイ、チーム分離 | API契約の同期コスト、PR追跡困難 |

## 結果
web/ と api/ で言語・ビルドツールを完全分離しつつ、同一PRで関連変更をまとめられる。Makefile で統合ビルド・テストを提供。将来規模が大きくなった場合はマルチレポへの移行も可能。
